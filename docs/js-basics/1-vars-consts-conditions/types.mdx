# 1.3 Типы данных

## Знания

:::tip[Что это такое?]
В JavaScript ___тип данных___ определяет вид и характеристики значения, которое хранится в переменной или константе.
Основные типы данных, которые нужно знать на текущий момент: `строка`, `число` и `булевое значение`.
:::

:::tip[Константы и переменные]
Не важно в константу устанавливается значение какого-то типа, или в переменную. Константы и переменные в этом плане работают одинаково.
:::

### **Строка (`string`)**

Ты уже успела создать переменную и константу, содержащие строки. Строка используется для хранения текста. Она заключается в одинарные `' '`, двойные `" "` или обратные кавычки `` ` ` ``.

Чем они отличаются, ты узнаешь постепенно. Пока, используй одинарные кавычки.

**Пример создания строки:**

```javascript
let greeting = 'Привет, мир!'; // Одинарные кавычки
const farewell = "До свидания!"; // Двойные кавычки
let message = `Сегодня отличный день!`; // Обратные кавычки (где знак Тильда в верхнем левом углу на клавиатуре)
```

### **Число (`number`)**

Числовой тип данных используется для хранения как целых, так и дробных чисел.

**Пример создания числа:**

```javascript
let age = 25; // Целое число
let price = 19.99; // Дробное число
```

### **Булевое значение (`boolean`)**

Boolean представляет логическое значение: `true` (истина) или `false` (ложь). То есть, всего 2 варианта.

Часто используется в условных операторах и циклах, до которых мы дойдем позже.

**Пример создания boolean:**

```javascript
let isStudent = true; // Истина
let hasLicense = false; // Ложь
```

:::danger[Не путай типы данных (но к этому нужно привыкнуть)]
```javascript
let isStudent = true; // это boolean
let age = 25; // это число
let hasLicense = "false"; // это строка, т.к. используются кавычки
age = "34"; // теперь это тоже строка
```
:::

:::warning[Особенности типов данных]
- Тип данных определяется значением, присвоенным переменной или константе.
- В JavaScript можно изменять тип данных переменной, присваивая ей новое значение другого типа.
- Используй правильный тип данных для повышения читаемости и предотвращения ошибок в коде. Например, не стоит писать число в кавычках, если только ты не делаешь это для какой-то конкретной цели.
:::
